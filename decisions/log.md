[2026-05-25] DECISION: Migrated RealTechee 2.0 frontend (www.realtechee.com + ai.realtechee.com) from AWS Amplify hosting to home server (`hetzim.ddns.net`, Docker + NPM proxy). Kept Amplify Gen2 backend on AWS (Cognito + AppSync + DynamoDB + S3 — all Always-Free at zero traffic). Stripped 5 Lambda functions from `amplify/backend.ts` and disabled the `postConfirmation` auth trigger because the CDK ESM assembler tripped on their `resource.ts` exports. Patched `utils/amplifyAPI.ts` to replace a hybrid Gen1/Gen2 `Amplify.configure` shape with the raw `amplify_outputs.json` (the hybrid form broke the Gen2 typed-client model registry — admin CRUD writes silently failed). Mid-migration `aws amplify delete-app` cascade-deleted the production backend (43 DDB tables + AppSync + Lambdas); recovered 2,672 of 2,673 items from local `./backups/migrations/*.json` (Aug 2025 snapshot) via `ampx sandbox --identifier restore` + boto3 `batch_write_item` import. Customer images (2.06 GB) survived in S3 due to DeletionPolicy:Retain. | REASONING: AWS 12-month free-tier expiring; business is closed so no active users; home server already hosts n8n/ollama/openwebui/supabase. Full Supabase migration would have required rewriting Amplify GraphQL client → Supabase client, which Doron explicitly excluded. Backend-on-AWS + frontend-on-home preserves the codebase as-is and lands at $0–$0.05/mo with zero code rewrites beyond the two unblocking edits (backend.ts strip + amplifyAPI.ts config fix). | CONTEXT: `.planning/STATE-FINAL.md`, `.planning/RESTORE-SUMMARY.md`, `.planning/MIGRATION-BRIEF.md`, memory `[[project-sunset-state]]`, `[[aws-amplify-delete-app-cascades-backend]]`, `[[restore-after-amplify-delete-app]]`, `[[runtime-endpoints]]`.
