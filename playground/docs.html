üéØ API Overview

The RealTechee GraphQL API provides a complete backend solution for managing real estate renovation projects, from initial customer requests to project completion. Built on AWS Amplify Gen 2 with DynamoDB, it offers powerful relationship management and real-time notifications.

Key Features

  • Lead Management: Complete customer request lifecycle from form submission to project conversion
  • Contact Management: Unified contact system with role-based assignments and preferences
  • Project Tracking: End-to-end project management with milestones, comments, and payment tracking
  • Quote Generation: Dynamic quote creation with itemized pricing and approval workflows
  • Notification System: Signal-driven notifications with email/SMS delivery and template management
  • Audit Trail: Comprehensive logging of all data changes with user attribution
üí° Pro Tip: Use the GraphiQL IDE for interactive exploration of the schema. All queries support filtering, sorting, and pagination out of the box.

üè¢ Core Business Entities

The RealTechee system is built around these primary entities, each representing a key aspect of the renovation business process.

Contacts
Core Entity
Central contact management with role-based assignments

Purpose: Unified contact system for homeowners, agents, and team members with role-based access control and notification preferences.

Key Fields

  • fullName String Complete contact name for display
  • email Email * Primary email address (unique identifier)
  • roleType String Business role: 'AE', 'PM', 'Admin', 'Customer', etc.
  • assignmentPriority Integer Priority for automatic assignments (1 = highest)
  • emailNotifications Boolean Email notification preference
  • smsNotifications Boolean SMS notification preference

Relationships

  • agentProjects - Projects where contact is the agent
  • homeownerProjects - Projects where contact is the homeowner
  • agentQuotes - Quotes managed by this agent
  • homeownerQuotes - Quotes for this homeowner
Requests
Lead Management
Customer service requests and lead tracking

Purpose: Manages the complete lead lifecycle from initial customer inquiry through qualification and conversion to projects.

Enhanced Case Management Fields

  • priority Enum low | medium | high | urgent
  • readinessScore Integer 0-100 score for quote readiness
  • estimatedValue Float Potential project value
  • followUpDate DateTime Next scheduled follow-up
  • missingInformation JSON Array of missing information items

Case Management Relationships

  • notes - RequestNotes for tracking communication
  • assignments - RequestAssignments for workload management
  • statusHistory - Complete status change audit trail
  • informationItems - Structured information gathering
  • scopeItems - Detailed project scope definition
Projects
Core Business
Active renovation projects with full lifecycle management

Purpose: Central project management entity with comprehensive tracking of financial, timeline, and stakeholder data.

Financial Tracking

  • budget String Project budget allocation
  • boosterEstimatedCost Float Estimated renovation costs
  • boosterActualCost Float Actual costs incurred
  • revShareAmount Float Revenue sharing amount

Multi-Contact Support

  • agent Contact Primary agent relationship
  • homeowner Contact Primary homeowner
  • homeowner2 Contact Secondary homeowner (spouse/partner)
  • homeowner3 Contact Additional stakeholder
Notification System
Signal-Driven
Real-time notification delivery with template management

Purpose: Complete notification infrastructure with signal emission, template management, and multi-channel delivery.

Core Components

  • SignalEvents: All emitted business events (form submissions, status changes)
  • SignalNotificationHooks: Configuration mapping signals to notifications
  • NotificationTemplate: Email/SMS template management
  • NotificationQueue: Delivery queue with retry logic
  • NotificationEvents: Comprehensive delivery logging
‚úÖ Production Ready: The notification system is fully operational with 100% delivery tracking and automatic retry functionality.

üöÄ Performance Guidelines

Best practices for optimal API performance and scalability.

Query Optimization

  • Use Pagination: Always specify limit for list operations (max 100 items)
  • Selective Field Loading: Only request fields you need to reduce payload size
  • Batch Related Data: Use relationship loading instead of multiple queries
  • Filter Early: Apply filters at the database level, not in client code
  • Use Indexes: Filter by indexed fields (id, email, status, createdAt)
